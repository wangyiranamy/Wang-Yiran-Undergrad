Source Code for Module zen.io.bel

  1  """ 
  2  The ``zen.io.bel`` module (available as ``zen.bel``) supports the reading and writing of edge lists in binary format.  The binary format was designed 
  3  by Derek Ruths for use with Zen.  In a binary format, each node is referenced by an index number (node/edge 
  4  labels and properties are not supported).  The complete format is: 
  5   
  6  ``<F = Format Version #><N = # of nodes> <M = # of edges> <u1><v1><u2><v2>...<um><vm>`` where: 
  7           
  8          * ``F`` is a 8 bit number indicating the format version used,  
  9          * ``N`` is a 32 bit number indicating the max index number (nodes are indexed from zero to ``N``), 
 10          * ``E`` is a 64 bit number indicating the number of edges stored, and 
 11          * ``ui`` and ``vi`` are ``ceil(log2(N+1))`` bit numbers storing the index numbers of the left and right nodes for edge ``i``. 
 12           
 13  The current version of the binary edge list (.bel) format is 1.0. 
 14   
 15  Reading 
 16  ------- 
 17   
 18  Graphs can be read either from file-like objects using :py:func:`zen.io.bel.read` or from strings using :py:func:`zen.io.bel.read_str`. 
 19  Both functions accept the same set of arguments: 
 20   
 21          * ``read(fh,<keyword arguments>)`` 
 22          * ``read_str(sbuffer,<keyword arguments>)`` 
 23   
 24  ``fh/sbuffer`` specifies the object from which the graph will be read. 
 25   
 26  Supported keywords include the following: 
 27   
 28          * ``node_obj_fxn [= str]``: unlike the default definition, this function accepts integers and returns the node object 
 29           
 30          * ``directed [= False]`` (boolean): indicates whether the data is read as directed 
 31           
 32          * ``check_for_duplicates [= False]`` (boolean): applies only when loading an undirected graph. If True, then a check will be made to ensure that 
 33            no duplicate edges are attempted to be added (in case the underlying graph was originally directed).  Checking incurs a small 
 34            performance cost due to the check. 
 35   
 36  .. autofunction:: zen.io.bel.read(fh[,node_obj_fxn=str,directed=False]) 
 37   
 38  .. autofunction:: zen.io.bel.read_str(fh[,node_obj_fxn=str,directed=False]) 
 39   
 40  Writing 
 41  ------- 
 42   
 43  Graphs can be written either to file-like objects using :py:func:`zen.io.bel.write` or to strings using :py:func:`write_str`. 
 44  Both functions accept the similar sets of arguments: 
 45   
 46          * ``write(G,fh)`` 
 47          * ``write_str(G)`` 
 48   
 49  ``G`` specifies the graph to write. ``fh`` is a file-like object.  ``write_str(...)`` returns the string 
 50  representation of the ``G``, so no argument equivalent of ``fh`` exists). 
 51   
 52  ``G`` must be compact (an exception will be raised if the graph is not compact).  The graph is stored in  
 53  the binary edge list format and node indexes are used to index the edge list when writing. 
 54   
 55  .. autofunction:: zen.io.bel.write(G,fh) 
 56   
 57  .. autofunction:: zen.io.bel.write_str(G)
