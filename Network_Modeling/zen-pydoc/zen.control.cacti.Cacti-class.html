Class Cacti

source code

The Cacti class represents a wrapper around a zen DiGraph and provides calculation of control related properties of the network. It can be used to find the location (and number) of inputs required to control the network as well as the underlying cacti structure. It can also be used to find the extent to which a network can be controlled by a specified set of controls and the related constrained cacti.

Method Details [hide private]

_build_cacti(self)

source code 

Computes a list of stems and a list of cycles comprising the cacti. Also computes the matched edges (matched in the maximum matching). If forced controls have been specified, it uses a maximum matching constrained by the specified controls.

controls_(self)

source code 

Returns the controls currently used in the cacti. If no controls were previously specified, this returns a list of tuples, each tuple indicating which node indices each control drives. If no controls were previously specified then this is the minimum controls required to control the entire network.

dicons_(self)

source code 

Returns the node indices that must be DIrectly CONtrolled (DICON) in this cacti accoring to the current controls. Essentially combines the contents of all the tuples of controls_() together into one list. If no controls were previously specified then these are the minimum dicons required to control the entire network.

dilations(self)

source code 

Returns the dilation sets (see the dilations_ function). The only difference is that the node objects are in the sets rather than the node indices.

num_controllable_nodes(self)

source code 

Returns the number of nodes that are controllable with the current controls. Calls generic_rank_ directly using the current controls.

num_controls(self)

source code 

Returns the number of controls corresponding to the cacti. If no controls were previously specified, this is the number of controls required to control the entire network.

num_dicons(self)

source code 

Returns the number of nodes that must be directly controlled in this cacti. A control can drive one stem and any number of cycles. If no controls were previously specified, then this is the number of nodes that must be directly controlled in order to control the network.

set_controls_(self, forced_controls)

source code 

Set the controls of the network with a list of tuples. Each tuple indicates which nodes (by index) that control is directly connected to. Erases any cacti already generated, but keeps the minimum controls if they have already been generated.
