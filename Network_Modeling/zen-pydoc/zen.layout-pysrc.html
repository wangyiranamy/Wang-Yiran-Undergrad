  1  """ 
  2  The ``zen.layout`` module provides functions for assigning positions to nodes and paths to edges according to a 
  3  selected algorithm.  Different algorithms attempt to maximize the aesthetic quality (e.g., readability) of the resulting 
  4  layout in different ways. 
  5   
  6  In order to standardize the handling of the graph visualization pipeline, all graph layout functions produce a new or  
  7  modify an existing :py:class:`zen.View` object. Graph layout functions all return the view object produced or modified. 
  8   
  9  **Standard arguments**: 
 10   
 11  *Positional arguments.* In addition to algorithm-specific parameters, each layout function accepts one positional argument: a graph or view object. 
 12  If a graph is specified, then a new view is created.  If a view is specified, then the graph of the view (:py:meth:`zen.View.graph`) 
 13  is used and the current layout of the view is replaced by the newly generated layout. 
 14   
 15  *Keyword arguments.* All layout functions also accept the following keyword arguments: 
 16   
 17          * ``bbox [=(0,0,100,100)]`` (:py:class:`tuple`): a tuple (left,bottom,top,right) indicating the spatial bounds that constrian the layout. 
 18            If set to ``None``, then positions and paths are unconstrained.  By default, the bounding box is (0,0,100,100). 
 19
