  10  of bit arrays and for logical operations on such arrays.  The 
  11  core idea used in this Python script for bin packing is based on 
  12  an internet posting by Josiah Carlson to the Pyrex mailing list.

  50  Constructing Bit Vectors 
  51  ------------------------ 
  52   
  53   You can construct a bit vector in six different ways. 
  54   
  55   (1) You can construct a bit vector directly from either a tuple 
  56       or a list of bits, as in 
  57   
  58          bv =  BitVector( bitlist = [1,0,1,0,0,1,0,1,0,0,1,0,1,0,0,1] )  
  59   
  60   (2) You can construct a bit vector from an integer by 
  61   
  62          bv =  BitVector( intVal = 56789 ) 
  63   
  64       The bits stored now will correspond to the binary 
  65       representation of the integer.  The resulting bit vector is 
  66       the shortest possible bit vector for the integer value 
  67       supplied.  For example, when intVal is 0, the bit vector 
  68       constructed will consist of just the bit 0. 
  69   
  70   
  71   (3) When initializing a bit vector with an intVal as shown 
  72       above, you can also specify a size for the bit vector: 
  73   
  74          bv = BitVector( intVal = 0, size = 8 ) 
  75   
  76       will return the bit vector consisting of the bit pattern 
  77       00000000.  The zero padding needed for meeting the size 
  78       requirement is always on the left.  If the size supplied is 
  79       smaller than what it takes to create the shortest possible 
  80       bit vector for intVal, an exception is thrown. 
  81   
  82            
  83   (4) You can create a zero-initialized bit vector of a given size 
  84       by 
  85   
  86          bv  = BitVector( size = 62 ) 
  87   
  88       This bit vector will hold exactly 62 bits, all initialized to 
  89       the 0 bit value. 
  90   
  91   (5) You can construct a bit vector from a disk file by a two-step 
  92       procedure. First you construct an instance of bit vector by 
  93   
  94          bv  =  BitVector( filename = 'somefile' )    
  95   
  96       This bit vector itself is incapable of holding the bits.  To 
  97       now create bit vectors that actually hold the bits, you need 
  98       to make the following sort of a call on the above variable 
  99       bv: 
 100   
 101          bv1 =  bv.read_bits_from_file( 64 )     
 102   
 103       bv1 will be a regular bit vector containing 64 bits from the 
 104       disk file. If you want to re-read a file from the beginning 
 105       for some reason, you must obviously first close the file 
 106       object that was acquired with a call to the BitVector 
 107       constructor with a filename argument.  This can be 
 108       accomplished by 
 109   
 110         bv.close_file_object() 
 111   
 112   (6) You can construct a bit vector from a string of 1's and 0's 
 113       by 
 114   
 115          bv  =  BitVector( bitstring = '110011110000' )       
 116   
 117   (7) Yet another way to construct a bit vector is to read the bits 
 118       directly from a file-like object, as in 
 119   
 120          x = "111100001111" 
 121          fileobj = StringIO.StringIO( x ) 
 122          bv = BitVector( fp = fileobj )

 129  Displaying Bit Vectors 
 130  ~~~~~~~~~~~~~~~~~~~~~~ 
 131   
 132  Since the BitVector class implements the __str__ method, a 
 133  bit vector can be displayed on a terminal by 
 134   
 135  :: 
 136   
 137          print bitvec 
 138   
 139  Basically, you can always obtain the string representation 
 140  of a bit vector by 
 141   
 142  :: 
 143   
 144          str( bitvec ) 
 145   
 146  and integer value by 
 147   
 148  :: 
 149           
 150          int( bitvec ) 
 151   
 152  Accessing and Setting Individual Bits and Slices 
 153  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ 
 154   
 155   2)  Any single bit of a bit vector bv can be set to 1 or 0 by 
 156   
 157             bv[M] = 1_or_0 
 158             print bv[M] 
 159   
 160       for accessing (and setting) the bit at the position that is 
 161       indexed M.  You can retrieve the bit at position M by bv[M]. 
 162   
 163   3)  A slice of a bit vector obtained by 
 164   
 165             bv[i:j] 
 166   
 167       is a bit vector constructed from the bits at index positions 
 168       from i through j-1. 
 169   
 170   4)  You can also carry out slice assignment: 
 171   
 172             bv1 = BitVector( size = 25 ) 
 173             bv2 = BitVector( bitstring = '1010001' ) 
 174             bv1[6:9]  = bv2[0:3] 
 175             bv3 = BitVector( bitstring = '101' )                  
 176             bv1[0:3]  = bv3 
 177   
 178       The first slice assignment will set the 6th, 7th, and  
 179       the 8th bits of the bit vector bv1 according to the first 
 180       three bits of bv2.  The second slice assignment will set 
 181       the first three bits of bv1 according to the three bits 
 182       in bv3. 
 183   
 184   5)  You can iterate over a bit vector, as illustrated by 
 185   
 186             for bit in bitvec: 
 187                 print bit,    
 188   
 189       This is made possible by the override definition for the 
 190       special __iter__() method. 
 191   
 192   6)  Negative subscripts for array-like indexing are supported. 
 193       Therefore, 
 194   
 195             bitvec[ -i ] 
 196   
 197       is legal assuming that the index range is not violated. 
 198   
 199   7)  You can reset a previously constructed bit vector to 
 200       either the all zeros state or the all ones state by 
 201   
 202             bv1 = BitVector( size = 25 ) 
 203             ... 
 204             ... 
 205             bv1.reset( 1 ) 
 206             ... 
 207             ... 
 208             bv1.reset( 0 ) 
 209   
 210       The first call to reset() will set all the bits of 
 211       bv1 to 1's and the second call all bit to 0's. 
 212   
 213  Logical Operations on Bit Vectors 
 214  ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~ 
 215   
 216   8) Given two bit vectors bv1 and bv2, you can perform bitwise 
 217      logical operations on them by 
 218   
 219             result_bv  =  bv1 ^ bv2 
 220             result_bv  =  bv1 & bv2 
 221             result_bv  =  bv1 | bv2 
 222             result_bv  =  ~bv1 
 223

 225  Comparing Bit Vectors 
 226  ~~~~~~~~~~~~~~~~~~~~~ 
 227   
 228   9) Given two bit vectors bv1 and bv2, you can carry out the 
 229      following comparisons that return Boolean values: 
 230   
 231             bv1 ==  bv2 
 232             bv1 !=  bv2 
 233             bv1 <   bv2 
 234             bv1 <=  bv2 
 235             bv1 >   bv2 
 236             bv1 >=  bv2 
 237   
 238      The equalities and inequalities are determined by the integer 
 239      values associated with the bit vectors.

 241  Other Supported Operations 
 242  ~~~~~~~~~~~~~~~~~~~~~~~~~~ 
 243   
 244  10)  You can permute and un-permute bit vectors: 
 245   
 246             bv_permuted   =  bv.permute( permutation_list ) 
 247   
 248             bv_unpermuted =  bv.unpermute( permutation_list ) 
 249   
 250   
 251  11)  Left and right circular rotations can be carried out by 
 252   
 253             bitvec  << N  
 254   
 255             bitvec  >> N 
 256   
 257       for circular rotations to the left and right by N bit 
 258       positions. 
 259   
 260   
 261  12)  A bit vector containing an even number of bits can be 
 262       divided into two equal parts by 
 263   
 264             [left_half, right_half] = bitvec.divide_into_two() 
 265   
 266        where left_half and right_half hold references to the two 
 267        returned bit vectors. 
 268   
 269   
 270  13)  You can find the integer value of a bit array by 
 271   
 272             bitvec.invValue() 
 273   
 274       or by 
 275   
 276             int( bitvec ) 
 277   
 278   
 279  14)  You can convert a bit vector into its string representation 
 280       by 
 281   
 282             str( bitvec ) 
 283   
 284   
 285  15)  Because __add__ is supplied, you can always join two 
 286       bit vectors by 
 287   
 288             bitvec3  =  bitvec1  +  bitvec2 
 289   
 290       bitvec3 is a new bit vector that contains all the 
 291       bits of bitvec1 followed by all the bits of bitvec2. 
 292   
 293         
 294  16)  You can write a bit vector directly to a file, as 
 295       illustrated by the following example that reads one bit 
 296       vector from a file and then writes it to another 
 297       file 
 298   
 299             bv = BitVector( filename = 'input.txt' ) 
 300             bv1 = bv.read_bits_from_file(64)         
 301             print bv1            
 302             FILEOUT = open( 'output.bits', 'wb' ) 
 303             bv1.write_to_file( FILEOUT ) 
 304             FILEOUT.close() 
 305             bv = BitVector( filename = 'output.bits' ) 
 306             bv2 = bv.read_bits_from_file( 64 ) 
 307             print bv2 
 308   
 309        IMPORTANT:  The size of bit vector must be a multiple of 
 310                    of 8 for this write function to work.  If this 
 311                    condition is not met, the function throws an 
 312                    exception. 
 313   
 314        IMPORTANT FOR WINDOWS USERS:  When writing an internally 
 315                    generated bit vector out to a disk file, it 
 316                    is important to open the file in the binary 
 317                    mode as shown.  Otherwise, the bit pattern 
 318                    00001010 ('\\n') in your bitstring will be written  
 319                    out as 0000110100001010 ('\\r\\n'), which is the 
 320                    linebreak on Windows machine. 
 321   
 322  17)  You can also write a bit vector directly to a stream 
 323       object, as illustrated by 
 324   
 325             fp_write = StringIO.StringIO() 
 326             bitvec.write_bits_to_fileobject( fp_write ) 
 327             print fp_write.getvalue()         # 111100001111  
 328         
 329   
 330  18)  You can pad a bit vector from the left or from the 
 331       right with a designated number of zeros 
 332   
 333             bitvec.pad_from_left( n ) 
 334   
 335             bitvec.pad_from_right( n ) 
 336   
 337       In the first case, the new bit vector will be the same 
 338       as the old bit vector except for the additional n zeros 
 339       on the left.  The same thing happens in the second 
 340       case except that now the additional n zeros will be on 
 341       the right. 
 342   
 343  19)  You can test if a bit vector x is contained in another bit 
 344       vector y by using the syntax 'if x in y'.  This is made 
 345       possible by the override definition for the special 
 346       __contains__() method. 
 347   
 348  20)  You can count the number of bits set in a BitVector  
 349       instance by 
 350   
 351         bv = BitVector( bitstring = '100111' ) 
 352         print bv.count_bits()                     # 4 
 353   
 354  21)  You can change the bit pattern associated with a 
 355       previously constructed BitVector instance: 
 356   
 357         bv = BitVector( intVal = 7, size =16 ) 
 358         print bv                              # 0000000000000111 
 359         bv.setValue( intVal = 45 ) 
 360         print bv                              # 101101

 363  How the Bit Vectors are Stored 
 364  ------------------------------ 
 365   
 366   The bits of a bit array are stored in 16-bit unsigned ints. 
 367   After resolving the argument with which the constructor is 
 368   called (which happens in lines (A2) through (A70) of the file 
 369   BitVector.py), the very first thing that the constructor does is 
 370   to figure out in line (A78) as to how many of those 2-byte ints 
 371   it needs for the bits.  For example, if you wanted to store a 
 372   64-bit array, the variable 'two_byte_ints_needed' in line (A78) 
 373   would be set to 4. (This does not mean that the size of a bit 
 374   vector must be a multiple of 16.  Any sized bit vectors can 
 375   constructed using the required number of two-byte ints.) Line 
 376   (A79) then creates an array of 2-byte ints and initializes it 
 377   with the required number of zeros.  Lines (A80) then shifts the 
 378   bits into the array of two-byte ints. 
 379   
 380   As mentioned above, note that it is not necessary for the size 
 381   of the vector to be a multiple of 16 even though we are using 
 382   C's unsigned short as as a basic unit for storing the bit 
 383   arrays.  The class BitVector keeps track of the actual number of 
 384   bits in the bit vector through the "size" instance attribute. 
 385   
 386   With regard to the code in lines (A2) through (A77) of the file 
 387   BitVector.py, note that, except for one case, the constructor 
 388   must be called with a single keyword argument, which determines 
 389   how the bit vector will be constructed.  The single exception to 
 390   this rule is for the keyword argument 'intVal' which can be used 
 391   along with the 'size' keyword argument.  When 'intVal' is used 
 392   with the 'size' option, the bit vector constructed for the 
 393   integer is the shortest possible bit vector.  On the other hand, 
 394   when 'size' is also specified, the bit vector is padded with 
 395   zeroes from the left so that it has the specified size. 
 396   
 397   Lines (A16) through (A22) are for the following sort of a call 
 398   
 399          bv = BitVector( filename = 'myfilename' ) 
 400   
 401   This call returns a bit vector on which you must subsequently 
 402   invoke the 'read_bits_from_file()' method to actually obtain a 
 403   bit vector consisting of the bits that constitute the 
 404   information stored in the file. 
 405   
 406   Lines (A23) through (A28) are for the case when you want to 
 407   construct a bit vector by reading the bits off a file-like 
 408   object, as in 
 409   
 410         x = "111100001111" 
 411         fileobj = StringIO.StringIO( x ) 
 412         bv = BitVector( fp = fileobj ) 
 413   
 414   Lines (A29) through (A61) are for the case when you want to 
 415   construct a bit vector from an integer, as in 
 416   
 417         bv = BitVector( intVal = 123456 ) 
 418   
 419   The bits stored in the bit vector will correspond to the binary 
 420   representation of the integer argument provided.  The bit vector 
 421   constructed with the above call will be the shortest possible 
 422   bit vector for the integer supplied.  As a case in point, when 
 423   the intVal is 0, the bit vector will consist of a single bit 
 424   which will be 0 also.  The code in lines (A27) through (A59) can 
 425   also handle the following sort of a call 
 426   
 427         bv = BitVector( intVal = 46, size = 16 )         
 428   
 429   which returns a bit vector of a specfic size by padding the 
 430   shortest possible bit vector the the intVal with zeros from the 
 431   left. 
 432    
 433   Lines (A62) through (A68) are for constructing a bit vector with 
 434   just the size information, as in 
 435   
 436         bv = BitVector( size = 61 ) 
 437   
 438   This returns a bit vector that will hold exactly 61 bits, all 
 439   initialized to the zero value. 
 440   
 441   Lines (A69) through (A73) are for constructing a bit vector from 
 442   a bitstring, as in 
 443   
 444         bv = BitVector( bitstring = '00110011111' ) 
 445   
 446   Finally, lines (A74) through (A77) are for constructing a bit 
 447   vector from a list or a tuple of the individual bits: 
 448      
 449         bv = BitVector( bitlist = (1, 0, 1, 1, 0, 0, 1) ) 
 450   
 451   The bit vector constructed is initialized with the supplied 
 452   bits.
