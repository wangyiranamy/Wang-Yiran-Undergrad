_end_current_token_then_expect(self, next_expected_token_type)

source code 

Ends the currently aggregating token, and puts it onto the list of tokens. Refreshes the tokenizers state to expect the next token next_expected_token_type should be string either 'key' or 'value'

tokenize(self, gml_string)

source code 

this is the main public method to be called. It accepts a gml string and returns a list of tokens. Tokens are tuples containing substrings, a token type (string) and the line number from which the token was taken.

In this method we see how the state of the tokenizer maps to behavior. processing is char by char, and the processing of a char is delegated to a private method based on the state of the tokenizer.
